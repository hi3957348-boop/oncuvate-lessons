# 업무정리 프로그램 — 개발계획서

> 윈도우용 원노트형 업무·법령/규정 정리 프로그램
> 버전: MVP v0.1 · 작성일: 2026-07-21

---

## 1. 목적 및 배경

업무 관련 법령·규정을 체계적으로 정리하고, 개정 이력을 추적하며, 세분화된 주제들이
서로 유기적으로 참조되는 관계를 한눈에 파악할 수 있는 **오프라인 데스크톱 프로그램**을 만든다.
원노트(OneNote)처럼 직관적인 계층 구조를 갖되, 상·하위 주제 구분이 명확하고
데이터를 표준 파일로 손쉽게 주고받을 수 있어야 한다.

## 2. 핵심 요구사항 → 기능 매핑

| # | 요구사항 | 구현 기능 |
|---|---------|----------|
| 1 | 변경 히스토리 파악 + 관련문서 첨부 | 조문별 **버전 타임라인**(개정 전후 비교) · **문서 첨부**(PDF/HWP/XLS를 버전에 연결) |
| 2 | 세분화·재결합되는 유기적 참조 | **양방향 상호 참조**(참조함/참조됨) · **호버 미리보기 팝업** |
| 3 | 윈도우 기반 + 수기 입력 용이 | Electron 데스크톱 앱 · 즉시 편집형 에디터 |
| 4 | 원노트형 + 상·하위 구조 명확 | **노트북 › 섹션 › 하위주제 › 페이지** 4단 계층 트리 · 색 구분 |
| 5 | 표준 파일 공유 + 간편 업데이트 | `.owp` 단일 파일 내보내기/가져오기 · 자동 업데이트 |

## 3. 기술 스택

| 영역 | 선택 | 사유 |
|------|------|------|
| 런타임 | **Electron** (Node.js 22 + Chromium) | 기존 HTML 프로토타입 UI를 그대로 이식, 오프라인 동작 |
| 언어 | HTML / CSS / JavaScript (Vanilla) | MVP 단계 의존성 최소화, 유지보수 단순 |
| 패키징 | **electron-builder** | 무설치 **Portable exe** + 설치형 동시 생성, 자동업데이트 지원 |
| 데이터 | JSON 기반 `.owp` 파일 | 사람이 읽을 수 있고 이식성 높은 표준 포맷 |
| 배포 | GitHub Releases | 자동 업데이트 피드로 활용 가능 |

## 4. 데이터 저장 방식 — 완전 포터블(Portable)

- **작업 데이터**: `업무정리.exe` **바로 옆 `data/` 폴더**에 저장 → USB째로 옮겨도 그대로 동작, PC에 흔적을 남기지 않음.
  - 개발 모드에서는 프로젝트 루트의 `data/` 폴더를 사용.
  - 운영(포터블 exe) 모드에서는 `실행파일_경로/data/` 를 사용.
- **워크스페이스 파일**: `data/workspace.owp` (전체 노트북·페이지·히스토리·참조·설정 포함), 편집 시 **자동 저장**.
- **공유 파일**: 「내보내기」로 임의 위치에 `.owp` 파일 저장 → 동료에게 전달. 「가져오기」로 불러오기.
- **첨부 문서**: 포터블 원칙에 따라 `data/attachments/` 폴더로 **복사 보관**(원본 이동/삭제와 무관하게 유지).

```
📁 USB드라이브/
 └ 📁 업무정리/
    ├ 업무정리.exe          ← 무설치 실행파일
    └ 📁 data/
       ├ workspace.owp      ← 모든 업무 데이터 (JSON)
       └ 📁 attachments/     ← 첨부 문서 사본
```

## 7. 빌드 및 배포

```bash
# 개발 실행
npm install
npm start

# 무설치 포터블 exe 빌드 (Windows 대상)
npm run build:win        # dist/ 에 업무정리-<버전>-portable.exe 생성
```

- **무설치 배포**: 생성된 `업무정리-portable.exe` 파일 하나만 전달하면 됨. 더블클릭 실행.
- **업데이트**: 새 버전 exe로 교체하거나(수동), Phase 3에서 electron-updater로 자동화.

## 9. 위험요소 및 대응

| 위험 | 대응 |
|------|------|
| Windows "알 수 없는 게시자" 경고 | 사내 배포는 무시 가능 / 필요 시 코드서명 인증서 |
| exe 용량(80~150MB) | 필요 시 Phase 3에서 Tauri 전환 검토 |
| 데이터 손상/실수 삭제 | 자동 저장 + 저장 시 직전 `.bak` 백업 |
| 여러 사람 동시 편집 | MVP는 파일 단위 공유(비동시) — 협업 편집은 향후 과제 |
